# JSONL data module (tags + events)

*This template includes sections for various types of tasks. Delete sections that don't apply to your specific use case.*

## Parent Initiative

[[CLO-I-0003]]

## Objective

Implement append-only JSONL storage for tags and events. These remain JSONL (not SQLite) because they're simple append-only data that benefits from git-friendly line diffs.

## Acceptance Criteria

## Acceptance Criteria

## Acceptance Criteria

- [ ] `TagStore` — reads/writes `data/tags.jsonl`
- [ ] `TagStore::add_tag(entity_id: &EntityId, tag: &Tag) -> Result<()>` — appends a line
- [ ] `TagStore::remove_tag(entity_id: &EntityId, tag: &str) -> Result<()>` — rewrites file excluding the entry
- [ ] `TagStore::get_tags(entity_id: &EntityId) -> Result<Vec<Tag>>`
- [ ] `TagStore::get_entities_by_tag(tag: &str) -> Result<Vec<EntityId>>`
- [ ] `EventStore` — reads/writes `data/events.jsonl`
- [ ] `EventStore::log(event: &Event) -> Result<()>` — appends a line
- [ ] `EventStore::read_all() -> Result<Vec<Event>>`
- [ ] `Event` struct: timestamp, event_type (created/updated/promoted/discarded/transitioned), entity_id, details (JSON)
- [ ] JSONL format: one JSON object per line, newline-terminated
- [ ] `cargo build` succeeds

## Implementation Notes

### Dependencies
Depends on: CLO-T-0019 (crate scaffold)

### Technical Approach
Simple file I/O with serde_json. Append = open in append mode, write line, flush. Read = BufReader line-by-line deserialize. Tag removal requires file rewrite (infrequent operation).
