## Objective

Integration tests covering full workspace lifecycle — init, write, read, search, federation, and sync coordination.

## Acceptance Criteria

## Acceptance Criteria

## Acceptance Criteria

- [ ] **Workspace init**: init creates correct directory tree, config files exist, open succeeds on initialized workspace
- [ ] **Content round-trip**: write markdown, read it back, delete it, confirm gone
- [ ] **Entity CRUD**: insert entity row, get it back, update, list by type/status/state, delete
- [ ] **Extraction lifecycle**: insert draft, list pending, promote (returns row), verify moved, discard another
- [ ] **JSONL tags**: add tags, get tags for entity, get entities by tag, remove tag
- [ ] **JSONL events**: log events, read all, verify chronological
- [ ] **FTS5 search**: index entities, search by keyword, verify results ranked, rebuild index from scratch
- [ ] **Federation**: ATTACH entities.db + relations.db, run cross-DB query, verify results
- [ ] **Sync coordination**: save entity via StoreSync, verify it appears in content + entities.db + graph + search index + event log
- [ ] **Temporal materialization**: save entity with deadline, verify HAS_DEADLINE graph edge was created
- [ ] All tests use temp directories (cleaned up after)
- [ ] All tests pass with `cargo test`

## Implementation Notes

### Dependencies
Depends on: CLO-T-0019 through CLO-T-0026 (all store modules)

### Technical Approach
Tests in `clotho-store/tests/store_tests.rs`. Each test creates a temp directory via `tempfile::tempdir()`, initializes a workspace, exercises the module, and lets the temp dir auto-clean.
