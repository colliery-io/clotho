## Objective

Implement `ContentStore` for reading, writing, and deleting markdown content files. Maps ContentBearing entities to filesystem paths under `.workspace/content/`.

## Acceptance Criteria

## Acceptance Criteria

- [ ] `ContentStore` struct taking workspace root path
- [ ] `write_content(entity_type: EntityType, id: &EntityId, content: &str) -> Result<PathBuf>` — writes markdown to correct subdirectory
- [ ] `read_content(entity_type: EntityType, id: &EntityId) -> Result<Option<String>>` — reads markdown if it exists
- [ ] `delete_content(entity_type: EntityType, id: &EntityId) -> Result<()>` — removes file
- [ ] `content_path(entity_type: EntityType, id: &EntityId) -> PathBuf` — resolves path without I/O
- [ ] `list_content(entity_type: EntityType) -> Result<Vec<PathBuf>>` — list all files in a subdirectory
- [ ] Path mapping: meetings/, reflections/, artifacts/, notes/, people/ subdirectories
- [ ] `cargo build` succeeds
