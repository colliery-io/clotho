## Objective

Implement `ExtractionStore` backed by SQLite (extractions.db) per CLO-A-0002. Manages draft entities pending human review with their extraction lifecycle.

## Acceptance Criteria

## Acceptance Criteria

## Acceptance Criteria

- [ ] `ExtractionStore` struct wrapping rusqlite::Connection to `data/extractions.db`
- [ ] `ExtractionStore::open(path) -> Result<Self>` — opens or creates with schema
- [ ] Schema: extractions table with: id TEXT PK, entity_type TEXT, title TEXT, speech_act TEXT, extraction_status TEXT, source_transcript_id TEXT, source_span_start INT, source_span_end INT, confidence REAL, created_at TEXT, metadata JSON
- [ ] `insert_draft(extraction: &ExtractionRow) -> Result<()>`
- [ ] `promote(id: &EntityId) -> Result<ExtractionRow>` — returns the promoted row (caller moves it to entities.db)
- [ ] `discard(id: &EntityId) -> Result<()>` — removes from extractions.db
- [ ] `list_pending() -> Result<Vec<ExtractionRow>>` — all drafts
- [ ] `list_by_confidence(min: f32) -> Result<Vec<ExtractionRow>>` — drafts above threshold
- [ ] `get(id: &EntityId) -> Result<Option<ExtractionRow>>`
- [ ] `ExtractionRow` struct matching the schema
- [ ] `cargo build` succeeds

## Implementation Notes

### Dependencies
Depends on: CLO-T-0019 (crate scaffold)

### Technical Approach
Separate from EntityStore — different database file, different lifecycle. promote() marks as Promoted and returns the row so the sync layer can insert it into entities.db and create graph nodes.
