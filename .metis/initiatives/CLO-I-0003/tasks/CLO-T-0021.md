# Entity data module (entities.db)

## Parent Initiative

## Objective

Implement `EntityStore` backed by SQLite (entities.db) per CLO-A-0002. Full CRUD for entity metadata with indexed queries by type, status, and state.

## Acceptance Criteria

## Acceptance Criteria

## Acceptance Criteria

- [ ] `EntityStore` struct wrapping rusqlite::Connection to `data/entities.db`
- [ ] `EntityStore::open(path) -> Result<Self>` — opens or creates entities.db with schema
- [ ] Schema: entities table with columns: id TEXT PK, entity_type TEXT, title TEXT, created_at TEXT, updated_at TEXT, status TEXT, task_state TEXT, extraction_status TEXT, source_transcript_id TEXT, source_span_start INT, source_span_end INT, confidence REAL, content_path TEXT, metadata JSON
- [ ] Indexes on entity_type, status, task_state, extraction_status
- [ ] `insert(entity: &EntityRow) -> Result<()>`
- [ ] `update(entity: &EntityRow) -> Result<()>`
- [ ] `get(id: &EntityId) -> Result<Option<EntityRow>>`
- [ ] `delete(id: &EntityId) -> Result<()>`
- [ ] `list_by_type(entity_type: EntityType) -> Result<Vec<EntityRow>>`
- [ ] `list_by_status(status: Status) -> Result<Vec<EntityRow>>`
- [ ] `list_by_state(state: TaskState) -> Result<Vec<EntityRow>>`
- [ ] `EntityRow` struct matching the schema (serializable bridge between domain entities and SQLite)
- [ ] `cargo build` succeeds

## Implementation Notes

### Dependencies
Depends on: CLO-T-0019 (crate scaffold)

### Technical Approach
In `src/data.rs` (or `src/data/entities.rs` if data.rs gets large). Uses rusqlite directly. EntityRow is a flat struct that maps 1:1 to the SQLite schema — conversion to/from domain entity types happens at a higher layer (sync).
