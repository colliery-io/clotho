# Refactor ContentStore for project-root paths

*This template includes sections for various types of tasks. Delete sections that don't apply to your specific use case.*

## Parent Initiative

[[CLO-I-0009]] — Visible Content Layout + Inbox

## Objective

Change `ContentStore` to resolve content paths relative to project root (parent of `.clotho/`) instead of `.clotho/content/`.

## Acceptance Criteria

## Acceptance Criteria

## Acceptance Criteria

- [ ] `ContentStore::new` takes `project_root` path (not workspace `.clotho/` path)
- [ ] `content_path()` returns `project_root.join(subdir).join(id.md)`
- [ ] All entity types map to correct visible directories: `programs/`, `responsibilities/`, `objectives/`, `workstreams/`, `tasks/`, `meetings/`, `reflections/`, `artifacts/`, `notes/`, `people/`
- [ ] Derived entities (Decision/Risk/Blocker/Question/Insight) get a `derived/` directory at root
- [ ] `write_content`/`read_content`/`delete_content`/`list_content` all use new paths
- [ ] `cargo build` succeeds
