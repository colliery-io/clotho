# SyncEngine: sync (add, commit, pull, push)

## Parent Initiative

[[CLO-I-0007]] — Git Sync Layer

## Objective

Implement the core sync flow: stage changes, commit, pull-rebase, push.

## Acceptance Criteria

## Acceptance Criteria

## Acceptance Criteria

- [ ] `sync()` method stages all files under `.workspace/` (`content/`, `data/`, `graph/`, `config/`)
- [ ] Checks for staged changes; returns early with `committed: false` if none
- [ ] Commits with timestamp message: `"clotho sync: YYYY-MM-DD HH:MM:SS"`
- [ ] If remote configured: pull with rebase before push
- [ ] Push to `origin/main`
- [ ] Returns `SyncResult` with `committed`/`pushed`/`files_changed`
- [ ] Handles no-remote gracefully (local-only commit)
- [ ] `cargo build` succeeds

## Status Updates

- sync() stages all files, checks for changes, commits with timestamp, pushes if remote
- Returns SyncResult{committed, pushed, files_changed}
- No-change detection via tree OID comparison with HEAD
- Fallback signature if git config not set ("clotho@localhost")
- push() uses ssh-agent credentials
- Handles first commit (no parent) and subsequent commits
