# SyncEngine: prune history

## Parent Initiative

[[CLO-I-0007]] — Git Sync Layer

## Objective

Implement history pruning to keep the workspace git repo small.

## Acceptance Criteria

## Acceptance Criteria

## Acceptance Criteria

- [ ] `prune_history(keep: usize)` squashes all commits older than the most recent `keep` into a single root commit
- [ ] Default `keep=20`
- [ ] Preserves HEAD state (working tree unchanged)
- [ ] Works with or without remote
- [ ] `cargo build` succeeds

## Implementation Notes

### Dependencies

- [[CLO-T-0048]] — SyncEngine: sync (add, commit, pull, push)

## Status Updates

- prune_history(keep) replays last `keep` commits on an orphan root
- Returns count of pruned commits (0 if total <= keep)
- commit_count() helper for verification
- Preserves HEAD tree state, only rewrites history
- `cargo build` succeeds
