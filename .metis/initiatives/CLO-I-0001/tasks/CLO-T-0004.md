## Objective

Define the 3 temporal traits (HasCadence, HasDeadline, HasSchedule) in `src/domain/traits.rs`. These model scheduling concerns as trait-based metadata on entities, with graph edge materialization for temporal queries.

## Acceptance Criteria

- [ ] `HasDeadline: Entity` — deadline() -> Option<DateTime<Utc>>, set_deadline(Option<DateTime<Utc>>)
- [ ] `HasSchedule: Entity` — scheduled_at() -> Option<DateTime<Utc>>, set_scheduled_at(Option<DateTime<Utc>>)

## Implementation Notes

### Dependencies
Depends on: CLO-T-0002 (Cadence/Frequency types), CLO-T-0003 (Entity trait)

### Technical Approach
These can go in the same traits.rs file as the entity traits, or a separate temporal.rs. Graph edge materialization (HAS_CADENCE, HAS_DEADLINE, HAS_SCHEDULE) is documented but implemented later in clotho-graph work.
