## Objective

Define the 4 LLM backend traits (Extractor, Summarizer, Resolver, Embedder) and their request/response/error types in `src/domain/llm_types.rs`. Per CLO-A-0001 and CLO-A-0003.

## Acceptance Criteria

## Acceptance Criteria

## Acceptance Criteria

- [ ] `Extractor` trait — async extract(ExtractionRequest) -> Result<ExtractionResult, ExtractorError>
- [ ] `Summarizer` trait — async summarize(SummaryRequest) -> Result<SummaryResult, SummarizerError>
- [ ] `Resolver` trait — async resolve(ResolutionRequest) -> Result<ResolutionResult, ResolverError>
- [ ] `Embedder` trait — async embed(&[&str]) -> Result<Vec<Vec<f32>>, EmbedderError>, dimension() -> usize
- [ ] All traits are `Send + Sync` (async_trait)
- [ ] Request types: ExtractionRequest, SummaryRequest, ResolutionRequest (with appropriate fields)
- [ ] Result types: ExtractionResult, SummaryResult, ResolutionResult (with appropriate fields)
- [ ] Error types: ExtractorError, SummarizerError, ResolverError, EmbedderError
- [ ] All types derive Serialize/Deserialize where appropriate
- [ ] `cargo build` succeeds

## Implementation Notes

### Dependencies
Depends on: CLO-T-0001 (scaffold), CLO-T-0002 (types — uses EntityId, ExtractionStatus, SourceSpan, etc.)

### Technical Approach
These are trait definitions only — no implementations. Concrete backends (Claude, Ollama) are in clotho-extract. Request/response types should be rich enough for the extraction pipeline but not coupled to any specific LLM provider.

## Status Updates

- Supporting types: SpeechAct (9-variant enum), Extraction, EntityMention, KnownEntity, ResolvedMention
