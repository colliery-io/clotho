# Scaffold clotho-tui crate with ratatui + crossterm

**Milestone**: 1 — App Shell + Embedded Terminal

## Objective

Create the `clotho-tui` crate in the workspace, wire up ratatui + crossterm dependencies, and get a minimal app loop rendering an empty shell. This is the foundation everything else builds on.

## Acceptance Criteria

## Acceptance Criteria

## Acceptance Criteria

- [ ] `clotho-tui` crate exists in workspace `Cargo.toml`
- [ ] Dependencies: `ratatui`, `crossterm`, `clotho-store`, `serde`, `serde_json`, `tokio`
- [ ] `clotho tui` subcommand launches the app (or standalone `clotho-tui` binary)
- [ ] App enters alternate screen, handles terminal resize, exits cleanly on `q` / `Ctrl-C`
- [ ] Basic event loop: renders a placeholder frame, processes key events

## Implementation Notes

### Technical Approach
- Add `clotho-tui/` alongside existing crates, add to workspace members
- Use `crossterm` backend for ratatui (cross-platform)
- Async event loop with `tokio` to support polling the store later
- Accept workspace path as CLI arg (default to current directory)

### Dependencies
- None — this is the first task in the initiative
