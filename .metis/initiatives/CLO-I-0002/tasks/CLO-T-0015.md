## Objective

Implement edge CRUD operations on GraphStore for adding, querying, and removing typed relations between nodes.

## Acceptance Criteria

## Acceptance Criteria

- [ ] `add_edge(&self, source: &EntityId, target: &EntityId, rel_type: RelationType) -> Result<(), GraphError>` — upserts edge with rel_type as the relationship label
- [ ] `add_edge_with_props(&self, source, target, rel_type, props: HashMap<String, Value>) -> Result<(), GraphError>` — for edges with properties (temporal edges carry date/cadence data)
- [ ] `remove_edge(&self, source: &EntityId, target: &EntityId, rel_type: RelationType) -> Result<(), GraphError>`
- [ ] `has_edge(&self, source: &EntityId, target: &EntityId, rel_type: RelationType) -> Result<bool, GraphError>`
- [ ] `get_edges_from(&self, source: &EntityId) -> Result<Vec<EdgeInfo>, GraphError>` — all outgoing edges
- [ ] `get_edges_by_type(&self, source: &EntityId, rel_type: RelationType) -> Result<Vec<EdgeInfo>, GraphError>`
- [ ] `get_edges_to(&self, target: &EntityId) -> Result<Vec<EdgeInfo>, GraphError>` — all incoming edges
- [ ] `EdgeInfo` struct: source_id, target_id, relation_type, properties
- [ ] RelationType maps to graphqlite relationship type string via sanitize_rel_type
- [ ] `cargo build` succeeds

## Implementation Notes

### Dependencies
Depends on: CLO-T-0013 (GraphStore), CLO-T-0014 (nodes — edges reference nodes)

### Technical Approach
In `src/graph/edges.rs`. Uses `graphqlite::Graph::upsert_edge()`, `delete_edge()`, `get_edge()`, `has_edge()`. RelationType enum's Display/string conversion maps to graphqlite relationship type strings. Edge queries use Cypher via `graph.query()`.

## Status Updates

- get_edges_from, get_edges_by_type, get_edges_to implemented via Cypher
- EdgeInfo struct: source_id, target_id, relation_type
- rel_type_to_string / parse_relation_type for RelationType ↔ string conversion
- Internal query_edges helper parses Cypher results into EdgeInfo

### graphqlite API notes
- upsert_edge props must be Vec<(String, String)> — AsRef<str> not HashMap<String, Value>
- has_edge/delete_edge take Option<&str> for rel_type
- No typed edge query API — must use Cypher via connection().cypher()
- type(r) in Cypher returns the relationship type string
