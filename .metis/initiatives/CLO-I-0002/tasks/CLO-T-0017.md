## Objective

Build common query helper methods on GraphStore for the query patterns defined in CLO-S-0005.

## Acceptance Criteria

## Acceptance Criteria

## Acceptance Criteria

- [ ] `get_neighbors(&self, id: &EntityId) -> Result<Vec<NodeInfo>, GraphError>` — all connected nodes
- [ ] `get_related_by_type(&self, id: &EntityId, rel_type: RelationType) -> Result<Vec<NodeInfo>, GraphError>` — outgoing edges of a specific type
- [ ] `get_incoming_by_type(&self, id: &EntityId, rel_type: RelationType) -> Result<Vec<NodeInfo>, GraphError>` — incoming edges of a specific type
- [ ] `get_entities_by_label(&self, entity_type: EntityType) -> Result<Vec<NodeInfo>, GraphError>` — all nodes of a type
- [ ] `raw_cypher(&self, query: &str) -> Result<CypherResult, GraphError>` — escape hatch for custom Cypher
- [ ] `stats(&self) -> Result<GraphStats, GraphError>` — node/edge counts
- [ ] `cargo build` succeeds

## Implementation Notes

### Dependencies
Depends on: CLO-T-0014 (nodes), CLO-T-0015 (edges)

### Technical Approach
In `src/graph/queries.rs`. Most methods use Cypher via `graph.query()` with parameterized queries where possible. `raw_cypher()` exposes the underlying graphqlite query method for CLI/MCP Cypher passthrough.

## Status Updates

- get_neighbors, get_related_by_type, get_incoming_by_type, get_entities_by_label implemented
- raw_cypher escape hatch for custom Cypher queries
- stats() returns GraphStats {node_count, edge_count}
- Internal query_nodes helper parses Cypher results into NodeInfo
- `cargo build` succeeds

### graphqlite API notes
- GraphStats fields are `nodes`/`edges` (not node_count/edge_count) and typed as i64 (not usize)
