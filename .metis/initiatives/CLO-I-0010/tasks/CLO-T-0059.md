## Objective

Create the `/daily-debrief` skill in the Clotho Claude Code plugin. This is the interactive entry point for the end-of-day ceremony — it gathers context, accepts materials, prompts for updates, checks the horizon, and launches the debrief-processor agent.

## Ceremony Flow

### Phase 1: Intake

| Step | Type | Action |
|------|------|--------|
| 1a | Tool | Scan `.clotho/inbox/` for unprocessed files — list what's accumulated |
| 1b | Tool | `clotho_list_entities` — filter to entities created today (check created_at) |
| 1c | Present | "Here's what I already have from today: [list]. Your inbox has N unprocessed files." |
| 1d | Tool | Ingest inbox files: `clotho_ingest` each one |
| 1e | Prompt | "Anything else from today? Drop transcripts, notes, or just tell me what happened." |
| 1f | Tool | For user-provided materials: `clotho_create_note` / `clotho_ingest` / `clotho_create_entity` |

### Phase 2: Status Update

| Step | Type | Action |
|------|------|--------|
| 2a | Tool | `clotho_list_entities(type=Task, state=doing)` + `(state=todo)` |
| 2b | Present | "Here are your active/pending tasks. What moved today?" (bulk list) |
| 2c | Prompt | User gives updates: "finished X, Y is blocked, started Z" |
| 2d | Tool | `clotho_update_entity` for each changed task |
| 2e | Prompt | "Any decisions, risks, or blockers from outside meetings?" |
| 2f | Tool | Create entities from user responses |

### Phase 3: Horizon Check

| Step | Type | Action |
|------|------|--------|
| 3a | Tool | Query tasks due this week (check deadlines) |
| 3b | Tool | Query tasks due next week |
| 3c | Tool | Query tasks in 'todo' state with old created_at (stale) |
| 3d | Tool | Query programs — check for programs with no recent task activity |
| 3e | Present | "Looking ahead: N items due this week, M next week. K tasks have been sitting in todo for >7 days. Program X has had no activity in 2 weeks." |
| 3f | Prompt | "Anything you want to reprioritize or flag?" |

### Phase 4: Launch Agent + Wrap

| Step | Type | Action |
|------|------|--------|
| 4a | Launch | Debrief-processor agent with today's unextracted content |
| 4b | Agent | Extracts, links, presents summary |
| 4c | Prompt | "Does this look right? Anything to adjust?" |
| 4d | Tool | `clotho_sync` |

## Implementation Notes

### Technical Approach
This is a SKILL.md file in the Claude Code plugin — not Rust code. It's a prompt that teaches Claude how to orchestrate the ceremony using the existing MCP tools. The skill uses `clotho_list_entities`, `clotho_search`, `clotho_ingest`, `clotho_create_entity`, `clotho_update_entity`, `clotho_create_relation`, `clotho_sync`, and launches the debrief-processor agent.
