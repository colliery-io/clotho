## Objective

Create the `debrief-processor` agent in the Clotho Claude Code plugin. This is the autonomous worker launched by the daily-debrief skill to extract structured entities from today's unprocessed content, build graph relations, and present results for review.

## Agent Processing Flow

| Step | Action | Tools Used |
|------|--------|-----------|
| 1 | Read each unextracted transcript/note from today | `clotho_read_entity` for each |
| 2 | Identify speech acts in content (Commit→Task, Decide→Decision, Risk→Risk, Block→Blocker, Question→Question, Insight→Insight) | Claude analysis (no tool — this is the LLM doing extraction) |
| 3 | Create derived entities for each speech act found | `clotho_create_entity` (type=decision/risk/blocker/question/insight/task) |
| 4 | Create EXTRACTED_FROM relations back to source transcript/note | `clotho_create_relation` |
| 5 | Identify people mentioned in content | Claude analysis |
| 6 | Create/find Person entities | `clotho_create_entity(type=person)` or `clotho_search` to find existing |
| 7 | Create MENTIONS relations from transcript → person | `clotho_create_relation` |
| 8 | Suggest BELONGS_TO relations to programs based on content context | `clotho_list_entities(type=Program)` to get available programs, then suggest links |
| 9 | Present extraction summary to user | Text output: "From today I extracted: N decisions, M risks, K tasks, J insights. Here they are: [list]" |

## Speech Act Reference

| Speech Act | Signal Patterns | Creates |
|------------|-----------------|---------|
| Commit | "I'll do X", "Let me handle" | Task |
| Decide | "We're going with X", "Decision is Y" | Decision |
| Risk | "The concern is...", "I'm worried about" | Risk |
| Block | "We're stuck on...", "Can't proceed until" | Blocker |
| Question | "We need to figure out...", "How do we..." | Question |
| Insight | "What we learned...", "Key takeaway..." | Insight |
| Delegate | "Can you take this?", "Assigning to X" | Task |
| Request | "I need X from you" | Task |
| Update | "Here's where we are..." | No entity (context only) |

## Implementation Notes

### Technical Approach
This is an agent .md file in the Claude Code plugin — not Rust code. It's a system prompt that teaches a subagent how to process transcripts using the speech act ontology and existing MCP tools. The agent is launched by the daily-debrief skill after Phase 2 completes.

### Key Design Decisions
- **Human-in-the-loop for relations**: Agent SUGGESTS BELONGS_TO links but does not create them automatically. The skill confirms with the user after the agent presents its summary.
- **All entities created directly**: No draft/promote lifecycle for in-session extraction. Entities are created as real entities (Decision, Risk, etc.) not as extraction drafts. The ceremony itself IS the review process.
- **Person dedup**: Agent should `clotho_search` for existing people before creating new ones to avoid duplicates.

### Dependencies
