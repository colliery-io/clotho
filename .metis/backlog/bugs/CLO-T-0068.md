# Entity lookup by prefix/short ID instead of full UUID

*This template includes sections for various types of tasks. Delete sections that don't apply to your specific use case.*

## Bug: Entity lookup by prefix/short ID

### Priority: P1

### Problem
All entity lookups (CLI `get`, `update`, `delete`, `relate`, `relations`, `ontology-get/set`, `processed-check/mark` and all MCP tools) require the full UUID. UUIDs are 36 characters and unusable by humans.

### Expected
Users should be able to use a prefix of the UUID (e.g., first 8 chars `f47ac10b`) and get the entity. If the prefix matches multiple entities, return all matches.

### Design

Add `resolve_id()` to EntityStore that returns a resolution enum:

```rust
enum ResolveResult {
    Exact(EntityRow),           // Full UUID matched
    Unique(EntityRow),          // Prefix matched exactly one
    Ambiguous(Vec<EntityRow>),  // Prefix matched multiple — SURFACE THIS
    NotFound,                   // Nothing matched
}
```

**Key principle: ambiguity always surfaces, never swallows.**

Keep `get()` as exact match (no breakage). Add `get_by_prefix()` for LIKE query. Add `resolve_id()` that tries exact first, then prefix.

### Caller behavior on Ambiguous

**CLI read-only** (`get`, `relations`, `ontology-get`, `processed-check`):
```
$ clotho get f47a
Ambiguous ID 'f47a' matches 3 entities:
  f47ac10b...  Program   "Monolith Breakup"
  f47a22de...  Task      "Write RFC"
  f47a9f01...  Note      "Architecture thoughts"
Use a longer prefix to narrow down.
```

**CLI destructive** (`update`, `delete`, `ontology-set`, `processed-mark`, `relate`, `unrelate`):
Same display, but refuses to proceed. Never mutate on ambiguous.

**MCP tools**: Return structured ambiguity response (not an error):
```json
{"status": "ambiguous", "prefix": "f47a", "matches": [...]}
```
The agent sees this and can pick the right one from context or ask the user.

### Where to fix
- `EntityStore`: add `get_by_prefix(prefix) -> Vec<EntityRow>` and `resolve_id(input) -> ResolveResult`
- All CLI commands that take entity ID: use `resolve_id()`, handle Ambiguous
- All MCP tools that take entity_id: use `resolve_id()`, return structured ambiguity
- GraphStore: add prefix-aware `has_node` / `get_node`
