# Background agent entity persistence failure

## Objective

Investigate and fix entity persistence failures when `clotho_create_entity` is called from a background Agent task.

## Backlog Item Details

### Type
- [x] Bug - Production issue that needs fixing

### Priority
- [x] P2 - Medium (nice to have)

### Impact Assessment
- **Affected Users**: Any user running background agent tasks that create entities
- **Reproduction Steps**: 
  1. Launch a background Agent task (via Claude Code background agent) that calls `clotho_create_entity`
  2. Wait for the agent task to report completion
  3. Query for the entity that was supposedly created
- **Expected vs Actual**: Expected: entity exists in the workspace after agent completes. Actual: entity does not persist. A background Agent task to compile an 8-week reflection "completed" successfully but the entity it created was missing. The agent had to recreate it manually.

### Likely Cause

Workspace session or database connection issue when running in background mode. The MCP server may not maintain a stable connection for background tool calls, or the transaction may not commit before the background process exits.

## Status Updates

### Investigation (2026-03-21)

Reviewed `workspace_resolver.rs` — workspace path is process-global via `Lazy<Mutex<Option<String>>>`. All agents connecting to the same MCP server process share the same workspace state. The global state is not the issue.

**Most likely cause:** Background agents in Claude Code may spawn their own MCP server process (separate from the foreground agent's server). If the background agent's MCP server auto-detects the workspace correctly, entity creation would work. But if the background agent runs in a different cwd or the MCP server lifecycle doesn't match the agent lifecycle, the entity could be created but the database connection dropped before flush.

**Alternatively:** The entity WAS created successfully but the background agent's result message was lost or the user queried for it with incorrect parameters (wrong title, wrong type filter).

**Verdict:** Cannot reproduce or fix without a controlled test case. This is likely a Claude Code platform behavior, not a clotho bug. Marking as blocked pending reproduction.
