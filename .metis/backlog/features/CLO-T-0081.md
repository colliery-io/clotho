# Store email addresses on Person entities

## Objective

Add an optional `email` field to Person entity metadata so calendar integrations and communication workflows can resolve email addresses without guessing.

## Backlog Item Details

### Type
- [x] Feature - New functionality or enhancement

### Priority
- [x] P2 - Medium (nice to have)

### Business Justification
- **User Value**: Calendar scheduling required 15+ tool calls with repeated email guessing (christina@ vs chrisheidt@, nico@ vs niko@). Person entities exist in the workspace but don't store contact information.
- **Business Value**: Enables reliable calendar integration and eliminates email guessing. Foundation for future communication workflow features.
- **Effort Estimate**: S

## Proposed Solution

Add optional `email` field to Person entity metadata. Support multiple emails (primary + aliases). CLI: `clotho create person --email "name@domain.com"`. MCP: include `email` in `create_entity` for type=person. Surface email in `read_entity` output. Allow updating via `update_entity`.
