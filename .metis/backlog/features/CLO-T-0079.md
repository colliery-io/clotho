# Extraction skill should prompt for batch size

## Objective

Make the extraction skill prompt the user for batch size when multiple unprocessed transcripts are detected, rather than attempting to process all of them at once.

## Backlog Item Details

### Type
- [x] Feature - New functionality or enhancement

### Priority
- [x] P2 - Medium (nice to have)

### Business Justification
- **User Value**: User invoked extraction with 19 unprocessed transcripts. The skill tried to extract all at once, overwhelming the user with questions. User had to course-correct: "doing all 19 at once will overwhelm me with questions; is there an iterative way we can do a transcript at a time?"
- **Business Value**: Better UX for first-time and batch-processing scenarios. Prevents user frustration and wasted context window.
- **Effort Estimate**: S

## Proposed Solution

When the extraction skill detects multiple unprocessed transcripts (threshold: >5), prompt the user: "Found N unprocessed transcripts. Process all at once, or one at a time?" Default to one-at-a-time for >5 transcripts. Allow the user to specify a custom batch size (e.g., "do 3 at a time").
