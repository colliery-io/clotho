# Persistent extraction skip patterns (configurable meeting type filters)

## Objective

Store extraction skip patterns in workspace configuration so the user does not have to re-state which meeting types to skip every session.

## Backlog Item Details

### Type
- [x] Feature - New functionality or enhancement

### Priority
- [x] P2 - Medium (nice to have)

### Business Justification
- **User Value**: User repeatedly told the agent to skip sport-team meetings, 1:1s during backfill, and DS Infra meetings. Had to re-state this preference each session. Preferences should be persistent.
- **Business Value**: Eliminates repeated skip instructions across sessions. Reduces user friction and context window waste.
- **Effort Estimate**: M

## Proposed Solution

Store skip patterns in workspace config (e.g., `.clotho/extraction-config.toml`). Config categories:
- `skip_titles` -- glob patterns matching transcript titles to skip (e.g., `"*Fantasy Football*"`, `"*1:1*"`)
- `skip_types` -- meeting type categories to skip

The extraction skill and debrief-processor should load these patterns before processing and auto-skip matching transcripts (with a log message noting the skip). Provide CLI commands to manage patterns: `clotho config skip-add "pattern"`, `clotho config skip-list`, `clotho config skip-remove "pattern"`.

- [ ] CLI commands to add, list, and remove skip patterns
- [ ] Skipped transcripts are still marked as processed (so they don't reappear in the unprocessed queue)
