# Auto mark_processed after extraction completes

## Objective

Automatically call `mark_processed` when extraction completes on a transcript, eliminating the manual bookkeeping step that the agent never skips.

## Backlog Item Details

### Type
- [x] Feature - New functionality or enhancement

### Priority
- [x] P2 - Medium (nice to have)

### Business Justification
- **User Value**: 42 manual `mark_processed` calls were observed in session 2, always immediately after extraction finishes on a transcript. The agent never skips this step -- it is pure mechanical overhead.
- **Business Value**: Eliminates 42+ unnecessary tool calls per extraction session. Removes a source of potential inconsistency if the agent forgets.
- **Effort Estimate**: S

## Proposed Solution

The debrief-processor agent and extraction skill should automatically call `mark_processed` when extraction completes successfully on a transcript. Alternatively, make the extraction operation itself mark the source as processed. Provide an `--no-auto-mark` flag for cases where the user wants manual control.
