# Compact summary output for batch entity/relation creation

## Objective

Return compact summary tables for batch operations instead of individual verbose confirmation blocks for each entity/relation created.

## Backlog Item Details

### Type
- [x] Feature - New functionality or enhancement

### Priority
- [x] P3 - Low (when time permits)

### Business Justification
- **User Value**: Creating 10 entities produces 10 near-identical markdown tables in the MCP response. This creates a wall of output that is hard to scan and wastes context window.
- **Business Value**: Cleaner output, easier review of batch operations. Pairs well with batch creation tools (CLO-T-0070, CLO-T-0073).
- **Effort Estimate**: S

## Proposed Solution

When multiple entities or relations are created in quick succession (or via batch tools), return a single condensed summary table: `| ID | Type | Title | Status |` rather than individual confirmation blocks. Apply this to both MCP tool responses and CLI output. Single-entity operations keep their current verbose format.
