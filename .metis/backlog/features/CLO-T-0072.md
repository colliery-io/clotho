# Entity content update tool (update_content or expand update_entity)

## Objective

Provide an API-level way to update entity body content, which `update_entity` currently cannot do (it only handles title/status/state).

## Backlog Item Details

### Type
- [x] Feature - New functionality or enhancement

### Priority
- [x] P1 - High (important for user experience)

### Business Justification
- **User Value**: `update_entity` only handles title/status/state -- not body content. Users created workaround "Program Context" notes as separate entities to hold content that should live on the entity itself. The agent used filesystem `Read`+`Edit` 7 times to modify content clotho owns, bypassing the API entirely.
- **Business Value**: Eliminates filesystem workarounds and the workaround-entity pattern that pollutes the knowledge graph.
- **Effort Estimate**: M

## Proposed Solution

Either (a) add a dedicated `clotho_update_content` MCP tool, or (b) add an optional `content` parameter to the existing `update_entity` tool. CLI: `clotho update <id> --content "new content"` or `clotho update <id> --content-file path`.

Option (b) is preferred for simplicity -- fewer tools to learn.
