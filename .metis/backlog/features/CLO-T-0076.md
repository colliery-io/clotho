# list_unprocessed tool for extraction queue

## Objective

Add a dedicated tool to query which transcripts/notes have not yet been extracted, enabling a reliable extraction queue workflow.

## Backlog Item Details

### Type
- [x] Feature - New functionality or enhancement

### Priority
- [x] P2 - Medium (nice to have)

### Business Justification
- **User Value**: No way to query "which transcripts haven't been extracted yet?" The user asked "do we need to re-process any?" and the Cypher query failed. The agent had to manually track extraction state, which is error-prone.
- **Business Value**: Enables reliable extraction queue workflow and supports the daily-brief "inbox" phase where unprocessed items need to be surfaced.
- **Effort Estimate**: S

## Proposed Solution

Add `clotho_list_unprocessed` MCP tool. Queries transcripts/notes that have no `processing_log` entry for "extraction". Returns a chronologically ordered list with entity ID, title, captured date. CLI: `clotho list-unprocessed` or `clotho inbox`.
