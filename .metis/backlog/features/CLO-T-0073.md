# Bulk directory capture tool

## Objective

Add a bulk directory capture tool that ingests all matching files from a directory in a single call, replacing the current pattern of repeated individual `clotho_capture` calls.

## Backlog Item Details

### Type
- [x] Feature - New functionality or enhancement

### Priority
- [x] P1 - High (important for user experience)

### Business Justification
- **User Value**: Ingesting 115 backfill transcripts took 12+ rounds of 10-at-a-time `clotho_capture` calls. Each file path had to be constructed individually with long Google Drive paths. This is extremely tedious for bulk ingestion.
- **Business Value**: Collapse 150+ capture calls into 1-2 calls for bulk ingestion scenarios. Critical for onboarding new workspaces with historical data.
- **Effort Estimate**: M

## Proposed Solution

Add `clotho_capture_directory` MCP tool and `clotho capture-dir` CLI command. Parameters: `path` (directory), `pattern` (glob, e.g. `*.md`), `entity_type` (e.g. `transcript`). Returns array of created entity IDs. Should handle duplicates gracefully (skip already-captured files based on filename or content hash).

## Acceptance Criteria

- [ ] `clotho_capture_directory` MCP tool accepts `path`, `pattern` (glob), and `entity_type`
- [ ] All matching files in the directory are captured as entities of the specified type

- [ ] CLI `clotho capture-dir <path> --pattern "*.md" --type transcript` works equivalently
- [ ] Duplicate files (already captured) are skipped with a warning, not errors
- [ ] Handles directories with 100+ files without timeout
