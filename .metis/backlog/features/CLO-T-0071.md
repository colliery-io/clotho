# create_note with optional parent_id auto-wiring

## Objective

Add an optional `parent_id` parameter to `clotho_create_note` so that notes can be created and linked to a parent entity in a single call.

## Backlog Item Details

### Type
- [x] Feature - New functionality or enhancement

### Priority
- [x] P2 - Medium (nice to have)

### Business Justification
- **User Value**: 100% of `create_note` calls (19/19 across sessions) are immediately followed by a `create_relation` to link the note to a parent. Two calls for what is conceptually one operation.
- **Business Value**: Eliminates 19+ unnecessary `create_relation` follow-up calls per session.
- **Effort Estimate**: S

## Proposed Solution

Add optional `parent_id` parameter to `clotho_create_note` (MCP tool and CLI). When provided, automatically create a `belongs_to` relation from the new note to the specified parent entity. The relation creation should happen atomically with the note creation.

## Acceptance Criteria

## Acceptance Criteria

- [ ] `clotho_create_note` accepts optional `parent_id` parameter
- [ ] When `parent_id` is provided, a `belongs_to` relation is auto-created from note to parent
- [ ] If `parent_id` references a non-existent entity, the entire operation fails (no orphan note)
- [ ] Omitting `parent_id` preserves existing behavior (note created with no relation)
- [ ] CLI `clotho create note --parent <id>` works equivalently
