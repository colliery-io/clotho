# read_entity with optional include_relations

## Objective

Add an `include_relations` option to `read_entity` so that entity data and its relations can be fetched in a single call instead of two.

## Backlog Item Details

### Type
- [x] Feature - New functionality or enhancement

### Priority
- [x] P3 - Low (when time permits)

### Business Justification
- **User Value**: `read_entity` + `get_relations` are always called as a pair (5+ occurrences per session). Two round-trips for what is conceptually one inspection operation.
- **Business Value**: Halves round-trips for entity inspection workflow. Small per-call savings but adds up across sessions.
- **Effort Estimate**: S

## Proposed Solution

Add optional `include_relations: true` parameter to `clotho_read_entity`. When set, include outgoing and incoming relations in the response alongside the entity data. Default to `false` for backward compatibility.
